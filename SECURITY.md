# Política de segurança

A segurança do MILU Software é tratada de forma responsável e privada.

## Como relatar uma vulnerabilidade

Utilize o recurso **Report a vulnerability** na aba **Security** deste repositório para abrir um aviso privado de segurança.

Inclua, quando possível:

- descrição objetiva do problema;
- impacto potencial;
- passos seguros para reprodução;
- recurso ou módulo afetado;
- evidências sem dados pessoais ou credenciais reais.

## Não faça

- não abra uma issue pública com detalhes exploráveis;
- não publique tokens, senhas, certificados ou dados de clientes;
- não execute testes destrutivos;
- não acesse, altere ou transfira dados que não sejam seus;
- não interrompa serviços ou integrações.

## Escopo

Este repositório contém documentação pública. Problemas relacionados ao produto principal também devem ser relatados privadamente, sem exposição do código ou da infraestrutura interna.

O recebimento de um relato não implica programa de recompensa financeira.
